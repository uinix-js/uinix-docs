It is a good practice to ensure that all SVG icons
- have consistent `width`, `height`, and `viewBox` properties
- have `fill` and `stroke` properties set to `'currentColor'`.

Doing the above allows icons to be configured with theme values (i.e. `sizes` and `colors`) when using with the [`ยง Icon`](#icon) component.

## `Icon`

Use the `Icon` component to easily render icons from the system.

```js live
<Layout spacing="m">
  <Icon icon="github.mark" />
  <Icon icon="up" />
</Layout>
```

If the relevant theme values are defined in `system.theme`,

```js
const system = {
  theme: {
    colors: {
      brand: {
        primary: '#0366d6',
        light: '#adb4bb',
      },
    },
    sizes: {
      icon: {
        s: '16px',
        m: '20px',
        l: '24px',
      },
    },
  },
},
```

You can easily apply appropriate sizes and colors to `Icon`.

```js live
<Layout spacing="m">
  <Icon
    color="brand.primary"
    icon="github.mark"
    size="icon.l"
  />
  <Icon
    color="brand.light"
    icon="up"
    size="icon.s"
  />
</Layout>
```

## `useIcon`

It is recommended to use the `Icon` component when building components with <PackageLink name="uinix-ui" />.  If you would simply like to retrieve the rendered SVG element, you can do this with the `useIcon` system hook.

```js live
() => {
  const Example = () => {
    const icon = useIcon('github.mark');
    return (
      <Layout align="center" spacing="s">
        {icon}
        <div>Github</div>
      </Layout>
    );
  }

  return <Example />
}
```

## Conclusion

We learned how to define SVG icons in `system.icons`, and configure them using the `Icon` component based on `system.theme` values.  We also learned how to use rendered icon elements with the `useIcon` system hook.
